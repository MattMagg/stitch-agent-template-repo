# Readiness Checklist

> Run through this checklist before starting a new design iteration or continuing to the next page.

## Pre-Design Readiness

- [ ] `DESIGN.md` exists and has been populated from the canonical screen
- [ ] `SITE.md` identifies the target surface for this iteration
- [ ] `next-prompt.md` has been written with a bounded task
- [ ] `metadata.json` has current Stitch project ID(s)
- [ ] Reference materials collected (if this iteration uses references)
- [ ] Previous surface approved and artifacts synced (if sequential)

## Pre-Implementation Readiness

- [ ] Target surface has an approved design in Stitch
- [ ] Screenshot saved to `.stitch/designs/images/`
- [ ] HTML exported to `.stitch/designs/html/` (if applicable)
- [ ] `DESIGN.md` is current — no unapproved drift
- [ ] Component patterns identified for extraction
- [ ] Implementation target directory ready (`src/`)
- [ ] Framework/tooling decisions documented in `SITE.md`

## Environment Readiness

- [ ] Stitch is accessible (via MCP or direct)
- [ ] Local repo is clean (no uncommitted work from previous iterations)
- [ ] Previous iteration's artifacts committed
